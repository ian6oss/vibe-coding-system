# 바이브코딩 시스템 오케스트레이터

## 역할

이 스킬은 개별 팀(기획/검수/개발/문서화)을 직접 수행하지 않는다.
대신 **전체 파이프라인의 흐름을 관리하고, 팀 간 데이터 전달을 조율**한다.

교향악단의 지휘자처럼 — 악기를 직접 연주하지 않지만, 누가 언제 연주할지를 결정한다.

## 전체 파이프라인

```
┌─────────────────────────────────────────────────┐
│              바이브코딩 시스템 (VCS)               │
│                                                  │
│  📝 기획    →  🔍 검수    →  🔨 개발    →  📚 문서화  │
│  (planner)    (reviewer)    (dev)       (docs)   │
│                                                  │
│  ←────── 오케스트레이터가 전체 흐름 관리 ──────→    │
└─────────────────────────────────────────────────┘
```

## 프로젝트 상태 추적

오케스트레이터는 프로젝트의 현재 상태를 항상 추적한다.

### 프로젝트 상태 파일
프로젝트 시작 시 상태 파일을 생성한다:
**파일명:** `[프로젝트명]_상태.md`

```markdown
# [프로젝트명] 프로젝트 상태

## 현재 단계: 🔍 검수 (Phase 2/4)

## 전체 진행률
[████████████░░░░░░░░] 40%

## 단계별 상태
| # | 단계 | 상태 | 산출물 | 시작 시간 |
|---|------|------|--------|----------|
| 1 | 📝 기획 | ✅ 완료 | 프로젝트_기획서_v1.1.md | 14:00 |
| 2 | 🔍 검수 | 🔄 진행 중 | (검수 리포트 생성 중) | 14:35 |
| 3 | 🔨 개발 | ⬚ 대기 | - | - |
| 4 | 📚 문서화 | ⬚ 대기 | - | - |

## 파일 추적
| 파일명 | 생성 단계 | 최종 수정 | 버전 |
|--------|----------|----------|------|
| 프로젝트_기획서_v1.0.md | 기획 | 14:20 | v1.0 |
| 프로젝트_기획서_v1.1.md | 기획 | 14:32 | v1.1 (검수 반영) |
| 프로젝트_검수리포트_v1.0.md | 검수 | (생성 중) | v1.0 |
```

## 단계 전환 로직

### 기획 → 검수
**전환 조건:**
- 기획팀이 기획서 초안 생성 완료
- 내부 검수 통과

**전달하는 것:**
- 기획서 파일 경로
- 내부 검수 결과

**사용자에게:**
> "📝→🔍 기획이 완료되어 검수팀에 넘깁니다."

### 검수 → (기획 수정 → 재검수) 또는 개발
**전환 조건:**
- 검수팀이 검수 리포트 생성 완료
- 사용자가 검수 결과를 확인

**분기:**
```
검수 등급 A or B → 사용자 확인 → 개발 단계로
검수 등급 C or D → 기획팀 수정 → 재검수
검수 등급 F → 기획 재시작 권고
```

**사용자에게:**
> "🔍→🔨 검수 통과! 개발팀에 기획서를 전달합니다."
또는:
> "🔍→📝 검수에서 주요 이슈가 발견되었습니다. 기획 수정 후 재검수합니다."

### 개발 → 문서화
**전환 조건:**
- 모든 빌드 블록 완성
- 최종 코드 리뷰 완료
- 통합 테스트 통과

**전달하는 것:**
- 완성된 코드 전체
- 기획서 (최종 버전)
- 검수 리포트
- 개발 과정의 대화 로그
- 코드 맵 (vibe-dev가 블록별로 업데이트한 것)

**사용자에게:**
> "🔨→📚 개발이 완료되었습니다! 문서화를 시작합니다."

### 문서화 완료
**산출물:**
- 프로젝트 레시피 (.md)
- 코드 맵 (.md)
- 유지보수 가이드 (.md)

**사용자에게:**
> "📚 문서화 완료! 프로젝트의 모든 과정이 기록되었습니다.
> 다음 프로젝트에서 레시피 파일을 참고하면 더 빠르게 만들 수 있습니다."

---

## 빠른 시작 모드

사용자가 "빨리 만들어줘" 또는 작은 프로젝트일 때:

```
전체 파이프라인:  기획(풀) → 검수(풀) → 개발(풀) → 문서화(풀)
속성 모드:        기획(속성) → 검수(간이) → 개발(풀) → 문서화(간이)
초속성 모드:      기획(3문답) → 검수(스킵) → 개발(풀) → 문서화(최소)
```

사용자에게:
> "프로젝트 규모에 맞는 모드를 선택해주세요:
> 1. **정식**: 기획부터 문서화까지 꼼꼼하게 (대규모 프로젝트)
> 2. **속성**: 핵심만 빠르게 (중규모 프로젝트)
> 3. **초속성**: 거의 바로 개발 (간단한 프로젝트)"

---

## 이전 프로젝트 레시피 활용

사용자가 "이 레시피 참고해서 새 프로젝트 만들어줘"라고 하면:

1. 레시피 파일을 읽는다
2. 이전 프로젝트의 기술 스택, 구조, 패턴을 파악한다
3. 새 프로젝트의 기획에서 이전 결정사항을 기본값으로 제안한다
4. 사용자가 바꾸고 싶은 것만 변경한다

> "이전 프로젝트 레시피를 확인했습니다.
> 기술 스택은 React + Express + PostgreSQL이었는데, 이번에도 같은 스택으로 갈까요?
> 아니면 바꾸고 싶은 부분이 있나요?"

## 이전 대화 이어가기 (세션 이어가기)

VCS는 대화가 끊겨도 이전 작업을 이어갈 수 있다.
"vcs 이어가기", "이어서 해줘", "어디까지 했어?" 등으로 트리거된다.

### 상태 저장 시스템

**매 단계 전환 시 자동으로 상태 파일을 저장한다.**

**파일 위치:** `docs/vcs/[프로젝트명]_상태.json`

### 이어가기 프로세스

사용자가 새 세션에서 "vcs 이어가기" 또는 "이어서 해줘"라고 하면:

1. `docs/vcs/` 에서 상태 파일을 찾는다
2. 여러 프로젝트가 있으면 목록을 보여주고 선택하게 한다
3. 상태를 읽고 요약을 보여준다:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔄 이전 작업 이어가기: [프로젝트명]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📍 마지막 위치: Phase 3/4 — 개발 (블록 5/12)
[██████████░░░░░░░░░░] 42%

📋 완료된 것:
  ✅ 기획서 v1.1 완성
  ✅ 검수 통과 (B등급)
  ✅ 블록 1~5 개발 완료

🔄 진행 중이던 것:
  → 블록 5: 장바구니 기능 (에러 핸들링 추가 필요)

📁 관련 파일:
  → 기획서: docs/vcs/쇼핑몰_기획서_v1.1.md
  → 검수: docs/vcs/쇼핑몰_검수리포트_v1.0.md
  → 코드맵: docs/vcs/쇼핑몰_코드맵.md

👉 이어서 진행할까요?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 중단 및 재개 (레거시 호환)

프로젝트가 중간에 중단되었을 때 상태 파일이 없는 경우:

1. `docs/vcs/` 에서 기획서, 검수 리포트 등 산출물 파일을 찾는다
2. 파일 존재 여부로 마지막 완료 단계를 추정한다
3. 그 다음 단계부터 재개한다

> "프로젝트 상태를 확인했습니다.
> 기획은 완료되었고, 검수까지 끝난 상태입니다.
> 개발 단계부터 재개할까요?"

## 다음 액션 추천 시스템

**매 단계 완료 시 반드시 다음 액션을 추천한다.** 사용자가 "다음에 뭐 해?"를 묻지 않아도, 각 단계가 끝나면 자동으로 안내한다.

### 추천 형식

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ [완료된 단계] 완료!
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📍 현재 위치: Phase N/4 완료
[████████████░░░░░░░░] N%

👉 다음 추천:
  "[다음 단계를 시작하는 자연어 명령]"
  예: "검수 시작해줘" / "개발 시작해줘" / "문서화해줘"

💡 또는:
  - "[대안 명령 1]" — [설명]
  - "[대안 명령 2]" — [설명]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 단계별 추천 내용

| 완료된 단계 | 다음 추천 (기본) | 대안 |
|---|---|---|
| vcs 시작 | "뭘 만들고 싶은지 말해주세요" | — |
| 기획 완료 | "검수 시작해줘" | "기획서 수정해줘" / "바로 개발해줘" |
| 검수 통과 (A/B) | "개발 시작해줘" | "기획서 좀 더 다듬어줘" |
| 검수 미통과 (C/D/F) | "기획 수정해줘" | "그래도 개발해줘" |
| 개발 완료 | "문서화해줘" | "코드 리뷰 한번 더" |
| 문서화 완료 | "수고했어! (프로젝트 종료)" | "레시피로 새 프로젝트 시작" |

### 블록 개발 중 추천 (vibe-dev 내부)

| 상황 | 추천 |
|---|---|
| 블록 N 완료 | "다음 블록 진행해줘" 또는 "이 블록 수정해줘" |
| 에러 발생 | "N번 수정안으로 해줘" 또는 "알아서 고쳐줘" |
| 전체 블록 완료 | "통합 테스트 돌려줘" → "문서화해줘" |
