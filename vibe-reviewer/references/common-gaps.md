# 바이브코딩 기획에서 자주 빠지는 것들

이 문서는 실제 바이브코딩 프로젝트에서 반복적으로 빠지는 항목들을 모은 것이다.
검수팀은 이 목록을 체크리스트처럼 활용하여 기획서의 빈 곳을 찾는다.

---

## 1. 거의 항상 빠지는 것 (95%+ 누락률)

### 에러 상황 처리
- 네트워크 끊김 시 어떻게 되는가?
- 서버 에러(500) 시 사용자에게 뭘 보여주는가?
- 입력 폼에서 유효하지 않은 데이터를 넣으면?
- 결제 도중 실패하면?
- 파일 업로드 실패 시?

### 빈 상태 (Empty State)
- 게시물이 하나도 없을 때 화면은?
- 검색 결과가 0건일 때?
- 장바구니가 비었을 때?
- 알림이 없을 때?

### 로딩 상태
- 데이터를 불러오는 동안 뭘 보여주는가?
- 스켈레톤 UI? 스피너? 프로그레스 바?
- 로딩이 5초 이상 걸리면?

### 권한/인증 예외
- 로그인하지 않은 사용자가 보호된 페이지에 접근하면?
- 세션이 만료되면?
- 권한 없는 기능에 접근하려 하면?

---

## 2. 자주 빠지는 것 (70%+ 누락률)

### 반응형 설계
- 모바일 레이아웃이 정의되지 않음
- 테이블이 모바일에서 어떻게 표시되는지 불명확
- 모바일 네비게이션 방식 미정의

### 개인정보/법적 사항
- 개인정보처리방침 페이지 누락
- 이용약관 페이지 누락
- 쿠키 동의 배너 누락
- GDPR/CCPA 등 지역별 법률 미고려

### 이메일/알림
- 회원가입 확인 이메일
- 비밀번호 재설정 이메일
- 주문 확인/배송 알림
- 이메일 템플릿 디자인

### 관리자 기능
- 사용자 관리 (차단, 삭제)
- 콘텐츠 관리 (수정, 삭제)
- 통계/대시보드
- 설정 관리

## 3. 종종 빠지는 것 (40%+ 누락률)

### 검색 기능 세부사항
- 검색어 자동완성
- 최근 검색어
- 검색 필터/정렬
- 검색 결과 페이지네이션

### 성능 관련
- 이미지 최적화 전략
- 캐싱 전략
- CDN 사용 여부
- 대용량 데이터 페이지네이션

### 다국어/국제화
- 텍스트 방향 (LTR/RTL)
- 날짜/시간 형식
- 통화 형식
- 번역 관리 방법

### 접근성
- 스크린 리더 지원
- 키보드 네비게이션
- 색상 대비
- alt 텍스트

### 데이터 백업/복구
- 백업 주기
- 복구 절차
- 데이터 보존 기간

## 4. 프로젝트 유형별 빠지기 쉬운 것

### 이커머스
- 재고 소진 시 처리
- 부분 환불 프로세스
- 쿠폰 중복 사용 정책
- 배송 불가 지역 처리
- 세금 계산 로직

### SaaS
- 구독 취소 후 데이터 처리
- 팀원 초대/제거 흐름
- 사용량 초과 시 처리
- 다운그레이드 시 기능 제한
- 데이터 내보내기(export) 기능

### 소셜/커뮤니티
- 신고/차단 시스템
- 스팸 방지
- 콘텐츠 모더레이션
- 알림 설정 (개별 on/off)

### 예약/일정
- 시간대(timezone) 처리
- 중복 예약 방지
- 취소/변경 정책
- 리마인더 발송

---

## 사용법

검수팀은 기획서를 받으면:
1. 프로젝트 유형을 파악한다
2. 이 문서의 해당 섹션을 참조한다
3. 기획서에 각 항목이 있는지 확인한다
4. 빠진 항목을 심각도와 함께 리포트에 포함한다

모든 항목이 반드시 기획서에 있어야 하는 것은 아니다.
"의도적으로 빠진 것"과 "깜빡한 것"은 다르다.
검수팀은 "이 항목이 기획서에 없는데, 의도적인가요?"라고 확인할 수 있다.
