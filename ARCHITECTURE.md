# 바이브코딩 시스템 (VCS: Vibe Coding System)

## 시스템 개요

바이브코딩의 3대 문제를 해결하는 통합 시스템:
1. **모호함** → 기획 에이전트팀이 디테일을 끝까지 파고듦
2. **품질 부재** → 개발 에이전트팀이 중간중간 검수
3. **유지보수 불가** → 문서화 시스템이 모든 것을 기록

## 핵심 설계: 기획팀과 검수팀의 독립 구조

기획팀(vibe-planner)과 검수팀(vibe-reviewer)은 **상하관계가 아닌 동등한 독립 팀**이다.

```
         ┌─────────────┐        ┌─────────────┐
         │  기획팀       │        │  검수팀       │
         │ (vibe-planner)│       │(vibe-reviewer)│
         │              │        │              │
         │ • 기획팀장    │        │ • 기술 검수관 │
         │ • 기획자      │        │ • UX 검수관   │
         │ • 마케터      │        │ • 비즈니스 검수관│
         └──────┬───────┘        └──────┬───────┘
                │                        │
                │  기획서 v1.0            │  검수 리포트 v1.0
                │                        │
                └───────┬────────────────┘
                        ↓
              ┌─────────────────┐
              │   사용자에게 동시 제시  │
              │                   │
              │ 📋 기획서          │
              │ 🔍 검수 리포트      │
              └────────┬──────────┘
                       ↓
                  사용자 결정
                       ↓
              ┌─ 수정 필요 → 기획팀 수정 → 검수팀 재검토 → 반복
              └─ 승인 → vibe-dev에 전달
```

**왜 2팀 독립 구조인가:**
- 자기가 만든 것의 빈 곳은 자기가 못 본다 (확인 편향 방지)
- 기획팀은 "만드는 데" 집중, 검수팀은 "부수는 데" 집중
- 서로 다른 관점(기술/UX/비즈니스)에서 교차 검증
- 의견 충돌 시 사용자가 최종 판단 (투명한 의사결정)

## 배포 형태

### Phase 1: Skill 세트 (현재)
- 각 단계를 독립 Skill로 제작
- Cowork + Claude Code에서 바로 사용 가능
- GitHub에 올려서 누구나 설치

### Phase 2: MCP 서버 추가
- 프로젝트 히스토리 관리 (Git 연동)
- 기획서/코드 버전 관리
- 프로젝트 간 지식 전달

### Phase 3: Plugin 번들링
- Skill + MCP를 하나의 .plugin 파일로 통합
- 원클릭 설치

## 핵심 워크플로우

```
사용자: "이민 관련 부자 사이트 만들어줘"
         ↓
[vibe-planner] 기획팀 가동
  ├── 기획팀장: 프로젝트 범위 설정, 질문 전략 수립
  ├── 기획자: 세부 기능별 질문 생성 및 수집
  └── 마케터: 시장 조사, 경쟁사 분석, 타겟 사용자 정의
         ↓
  → 기획서 초안 (spec_v1.0.md) 출력
         ↓
[vibe-reviewer] 검수팀 가동 (독립적으로)
  ├── 기술 검수관: 기술 스택/설계 건전성 검증
  ├── UX 검수관: 사용자 흐름/화면 상태 검증
  └── 비즈니스 검수관: 사업성/법적 요구사항 검증
         ↓
  → 검수 리포트 (review_v1.0.md) 출력
         ↓
[사용자에게 기획서 + 검수 리포트 동시 제시]
         ↓
  → 수정 반영 → 재검수 → 최종 승인
         ↓
[vibe-dev] 개발 에이전트팀 가동
  ├── 큰 기능 → 작은 기능으로 분할
  ├── 기능별 개발 + 중간 검수
  ├── 에러 발생 시 원인 분석 + 수정 방향 제안
  └── UI 미리보기 제공
         ↓
  → 완성된 코드 + 리뷰 결과
         ↓
[vibe-docs] 문서화 시스템 가동
  ├── 프롬프트 기록 (어떤 지시가 이 결과를 만들었나)
  ├── 코드 맵 (어떤 파일이 어떤 기능을 담당하나)
  ├── 기술 스택 정리 (무엇으로, 왜 이렇게 만들었나)
  └── 재사용 가능한 참조 문서 (.md) 생성
         ↓
  → 다음 프로젝트에서 "이 문서 참고해서 만들어줘" 가능
```
