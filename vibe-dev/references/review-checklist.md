# 코드 리뷰 체크리스트

코드 검수관이 매 블록 완성 시 사용하는 체크리스트.

---

## 블록별 리뷰 (매 블록 완성 시)

### 기능 정확성
- [ ] 기획서에 정의된 기능이 모두 구현되었는가?
- [ ] 기획서에 없는 기능이 임의로 추가되지 않았는가?
- [ ] 엣지 케이스가 처리되었는가?
- [ ] 에러 상황에서 적절한 피드백이 있는가?

### 코드 품질
- [ ] 변수/함수 이름이 의미를 전달하는가?
- [ ] 불필요한 코드(주석 처리된 코드, 미사용 import)가 없는가?
- [ ] 함수가 하나의 역할만 하는가? (Single Responsibility)
- [ ] 하드코딩된 값이 상수나 환경변수로 분리되었는가?
- [ ] 주석이 "왜"를 설명하는가? (코드 자체가 "무엇을"를 설명)

### 보안
- [ ] 사용자 입력이 검증(validation)되는가?
- [ ] SQL Injection 방지가 되어있는가? (Prepared Statement / ORM)
- [ ] XSS 방지가 되어있는가? (이스케이핑)
- [ ] 민감한 정보(API키, 비밀번호)가 코드에 하드코딩되지 않았는가?
- [ ] 인증이 필요한 API가 보호되어있는가?

### 성능
- [ ] 불필요한 리렌더링이 없는가? (React)
- [ ] N+1 쿼리 문제가 없는가? (DB)
- [ ] 대용량 데이터에 페이지네이션이 적용되었는가?
- [ ] 이미지가 최적화되었는가? (lazy loading, 적절한 포맷)

### 유지보수성
- [ ] 다른 사람(또는 미래의 바이브코더)이 이 코드를 읽고 이해할 수 있는가?
- [ ] 파일 구조가 직관적인가?
- [ ] 컴포넌트/함수가 재사용 가능한 형태인가?
- [ ] 매직 넘버(의미 불명의 숫자)가 없는가?

---

## 통합 리뷰 (전체 완성 시)

### 아키텍처
- [ ] 폴더 구조가 프로젝트 규모에 적절한가?
- [ ] 관심사 분리(Separation of Concerns)가 되어있는가?
- [ ] 순환 의존성이 없는가?
- [ ] 설정과 코드가 분리되었는가? (.env, config)

### 통합 동작
- [ ] 블록 간 데이터 흐름이 정상인가?
- [ ] 전체 사용자 시나리오가 끝까지 작동하는가?
- [ ] 라우팅이 모든 페이지를 커버하는가?
- [ ] 404/에러 페이지가 있는가?

### 배포 준비
- [ ] 환경변수가 정리되었는가? (.env.example 제공)
- [ ] 빌드가 에러 없이 완료되는가?
- [ ] README 또는 실행 방법이 문서화되었는가?
- [ ] 불필요한 console.log가 제거되었는가?

## 리뷰 등급 기준

| 등급 | 의미 | 기준 |
|------|------|------|
| A+ | 우수 | 모든 체크항목 통과 + 코드가 교과서적 |
| A  | 양호 | 주요 항목 모두 통과, 사소한 개선 1-2개 |
| A- | 수용 가능 | 기능 정상, 품질 개선 3-5개 |
| B  | 수정 필요 | 기능은 되지만 보안/성능 이슈 있음 |
| C  | 재작업 | 주요 기능 미구현 또는 심각한 결함 |
