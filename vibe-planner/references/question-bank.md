# 도메인별 질문 은행

이 문서는 프로젝트 유형별로 반드시 물어봐야 하는 질문들의 모음이다.
기획 에이전트는 사용자의 프로젝트 유형을 파악한 후, 해당 도메인의 질문을 기반으로 대화를 이끌어간다.

---

## 공통 질문 (모든 프로젝트)

### 1단계: 본질 파악 (What & Why)
- 이 프로젝트를 한 문장으로 설명한다면?
- 왜 이것을 만들려고 하는가? (개인 프로젝트 / 사업 / 학습 / 의뢰)
- 이것을 사용할 사람은 누구인가? (나만 / 특정 그룹 / 불특정 다수)
- 비슷한 서비스 중 참고하고 싶은 것이 있는가?
- 예산과 시간 제약이 있는가?

### 2단계: 범위 설정 (Scope)
- MVP(최소 기능)로 먼저 만들 것인가, 완성형으로 갈 것인가?
- 핵심 기능 3가지를 꼽는다면?
- 나중에 추가하고 싶은 기능은?
- 다국어 지원이 필요한가?
- 접근성(장애인, 고령자) 고려가 필요한가?

### 3단계: 기술 환경 (Tech Context)
- 이미 사용 중인 기술/도구가 있는가?
- 호스팅/배포 환경에 대한 선호가 있는가?
- 도메인(URL)이 있는가?
- 기존 데이터가 있는가? (엑셀, DB, API 등)

---

## 웹사이트/웹앱

### 구조 & 페이지
- 어떤 페이지들이 필요한가? (홈, 소개, 서비스, 연락처...)
- 각 페이지에서 사용자가 할 수 있는 행동은?
- 네비게이션 구조는 어떻게? (상단 메뉴, 사이드바, 햄버거)
- 푸터에 들어갈 정보는?

### 디자인 & UI
- 선호하는 느낌/분위기는? (깔끔, 화려, 미니멀, 고급, 친근)
- 주요 색상이나 브랜드 컬러가 있는가?
- 로고가 있는가?
- 참고하고 싶은 디자인의 사이트가 있는가?
- 다크모드 지원이 필요한가?
- 애니메이션/인터랙션 수준은? (없음 / 기본 / 풍부)

### 콘텐츠
- 텍스트 콘텐츠는 누가 작성하는가?
- 이미지/영상 소스는? (직접 촬영 / 스톡 / AI 생성)
- 블로그나 뉴스 섹션이 필요한가?
- CMS(콘텐츠 관리 시스템)가 필요한가?

### 기능
- 회원가입/로그인이 필요한가?
  - 소셜 로그인? (구글, 카카오, 네이버, 애플)
  - 이메일 인증?
- 결제 기능이 필요한가?
  - 어떤 결제 수단? (카드, 계좌이체, 카카오페이, 토스)
  - 구독 결제? 일회성 결제?
- 검색 기능이 필요한가?
  - 검색 범위는? (전체 / 특정 카테고리)
- 지도/위치 기능이 필요한가?
- 알림(이메일, 푸시, SMS)이 필요한가?
- 관리자 페이지(어드민)가 필요한가?

### 데이터 & 보안
- 개인정보 수집이 있는가?
  - 어떤 개인정보? (이름, 이메일, 전화번호, 주소)
- 개인정보처리방침/이용약관이 필요한가?
- SSL 인증서는?
- 데이터 백업 주기는?

### 성능 & 확장
- 예상 동시 접속자 수는?
- SEO(검색엔진 최적화)가 중요한가?
- 로딩 속도에 대한 기대치는?

## 질문 사용 가이드

이 질문 은행을 사용할 때의 핵심 원칙:

1. **한 번에 3-5개 질문만**: 사용자를 질문 폭탄으로 압도하지 않는다
2. **우선순위 순서로**: 프로젝트의 본질부터, 디테일은 나중에
3. **맥락에 맞게 선별**: 모든 질문을 다 하지 않는다. 프로젝트에 해당되는 것만
4. **모른다고 하면 제안**: "보통 이런 경우에는 ~을 많이 씁니다" 형태로 가이드
5. **답변을 기반으로 후속 질문**: 은행에 없는 질문도 대화 흐름에 따라 생성
6. **확인 질문**: "정리하면 ~인 건가요?"로 중간중간 확인
